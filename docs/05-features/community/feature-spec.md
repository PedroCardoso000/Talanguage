# Feature Spec - Community

## Objetivo
Preparar o modulo de comunidade para uma futura liberacao sem mentir para o usuario na versao atual.

## Contexto atual
A UI de comunidade existia como fluxo parcialmente funcional, mas a estabilizacao da primeira versao exige uma apresentacao honesta de "proxima versao".

## Escopo da versao atual
Inclui:
- rota `/community`;
- tela de preview bloqueado;
- explicacao clara do valor futuro;
- cards ofuscados/desabilitados;
- orientacao para os modulos ja funcionais.

## Fora de escopo
- listar grupos reais;
- listar parceiros reais;
- registrar interesse na interface atual;
- chat em tempo real;
- chamada de video;
- feed social;
- matching inteligente complexo.

## Criterio de pronto
- comunidade aparece como proxima versao de forma honesta;
- a UI nao promete funcionalidade inexistente;
- o usuario entende o que fazer agora dentro do produto.
