## Tarefa

Revisar:

- rotas;
- estados de loading/error/empty;
- mocks mortos;
- duplicação de componentes;
- regras de domínio no frontend;
- contratos divergentes;
- testes críticos;
- build/lint;
- documentação desatualizada.

## Regras

- Não criar features novas.
- Não redesenhar UI.
- Não trocar stack.
- Corrigir apenas problemas alinhados à primeira versão.

## Entrega esperada

Informe:

1. problemas encontrados;
2. correções feitas;
3. pendências;
4. validações executadas;
5. recomendação de próxima fase.
